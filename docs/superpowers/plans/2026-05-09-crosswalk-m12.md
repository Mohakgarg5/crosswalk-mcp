# Crosswalk M12 Implementation Plan — Smart fill (apply_application v1.3)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.3.0 by adding **form-introspection-then-sample** to `apply_application`. Before filling, the tool now calls `browser.preview(deepLink)` to extract the form's actual textareas. For each textarea whose `name` isn't already covered by the profile/answerPack/cover-letter pipeline, it calls the LLM (via sampling) to generate a 1-3 sentence answer using the application's tailored cover letter as context. Those generated answers are pushed as `text_by_name` fields keyed by the form's actual `name`. The tool still doesn't submit — review-then-click-Submit semantics unchanged.

**Architecture:**
- New helper `sampleAnswerForFormField({ sampling, formField, jobContext, applicantContext })` in `src/services/sampling/answerFormField.ts`. Builds a tight prompt asking the LLM for a brief answer, with a `SKIP` escape hatch for unanswerable questions. Returns `string | null`.
- `apply_application`'s `ctx` type expands from `{ db, browser }` to `{ db, browser, sampling }` — bringing it in line with the rest of the tools that consume the shared `ToolCtx`. The tool registration in `src/tools/index.ts` already passes the full `ToolCtx`, so no plumbing change there.
- Inside `applyApplication`, after the existing profile/cover-letter/answerPack field-building, we call `browser.preview(app.deepLink)`, filter `formFields[]` to type=`'textarea'` entries whose `name` isn't already in the pushed fields list and whose label doesn't look like a cover-letter slot, then call the sampling helper for each. Successful answers become `text_by_name` fields with the form's `name`.
- Trade-off: smart fill makes two browser calls per `apply_application` invocation (preview + fillForm), so wall-clock time roughly doubles to ~10s on real forms. Acceptable for a once-per-application operation that the user manually inspects after.

**Tech Stack:** Same as M11. No new dependencies.

**M12 ships:**
- `sampleAnswerForFormField` sampling helper
- `apply_application` now does form-introspection-then-sample for unmatched textareas
- Updated `ApplyApplicationResult` with new optional `sampledFields: string[]` reporting which form names received LLM-generated answers
- README + USER_GUIDE updates noting smart fill
- Version bump 1.2.0 → 1.3.0; tag + push

**Out of M12 (deferred):**
- **Per-ATS selector packs.** Greenhouse / Lever / Ashby / Workable have idiosyncratic patterns; a clean per-ATS architecture is its own task. v1.4/M13.
- **Workday widget support.** Workday's Lightning components use `data-automation-id` attributes; selectors are completely different. v1.5/M14.
- Sampling for non-textarea unmatched fields (selects, radios, multi-step). Out of scope for v1.3.
- Single-session optimization (combining preview + fill into one browser session). Doable but complicates the Browser interface; v1.4+ if performance becomes a real concern.
- Cover letters / Q&A textareas WHERE THE FORM'S `name` is unconventional and our coverage selectors don't catch — partially helped by smart fill, but a real fix needs per-ATS work.

- [ ] **Step 3: Implement `src/services/sampling/answerFormField.ts`**

```ts
import type { SamplingClient } from '../../sampling/client.ts';
import type { FormField } from '../browser/types.ts';

export type SampleAnswerOpts = {
  sampling: SamplingClient;
  formField: FormField;
  jobContext: string;
  applicantContext: string;
};

const SYSTEM = `You are helping a job applicant fill out a form question. \
Write a concise, sincere 1-3 sentence answer in the applicant's voice. \
Return ONLY the answer text — no preamble, no markdown, no quotes. \
If the question cannot be reasonably answered from the provided context (for example, it asks about a specific past experience that wasn't shared), respond with exactly the word "SKIP" and nothing else.`;

export async function sampleAnswerForFormField(opts: SampleAnswerOpts): Promise<string | null> {
  const { sampling, formField, jobContext, applicantContext } = opts;
  const label = formField.label || formField.name;
  const prompt = [
    `Form question: ${label}`,
    `Required: ${formField.required ? 'yes' : 'no'}`,
    '',
    `Job context: ${jobContext}`,
    `Applicant context: ${applicantContext}`,
    '',
    'Write the answer.'
  ].join('\n');

  let raw: string;
  try {
    raw = await sampling.complete({
      prompt,
      system: SYSTEM,
      maxTokens: 250,
      temperature: 0.4
    });
  } catch {
    return null;
  }

  const trimmed = raw.trim();
  if (trimmed === 'SKIP') return null;
  return trimmed;
}
```

```ts
  const sampledFields: string[] = [];
  const knownNames = new Set<string>();
  for (const f of fields) {
    if (f.kind === 'text_by_name') knownNames.add(f.name);
  }

  let preview: Awaited<ReturnType<Browser['preview']>> | null = null;
  try {
    preview = await ctx.browser.preview(app.deepLink);
  } catch {
    // If the preview call fails (rare — fillForm right after would also fail), skip smart fill.
    preview = null;
  }

  if (preview && preview.formFields.length > 0) {
    const job = getJob(ctx.db, app.jobId);
    const company = job ? getCompany(ctx.db, job.companyId) : null;
    const jobContext = job
      ? `${job.title}${company ? ` at ${company.name}` : ''}`
      : 'a job application';
    const applicantContext = (app.coverLetterMd && app.coverLetterMd.length > 0)
      ? app.coverLetterMd.slice(0, 1500)
      : 'No cover letter on file.';

    for (const formField of preview.formFields) {
      if (formField.type !== 'textarea') continue;
      if (!formField.name || knownNames.has(formField.name)) continue;
      const labelOrName = (formField.label || formField.name).toLowerCase();
      if (/cover[\s_-]?letter/.test(labelOrName)) continue;

      const answer = await sampleAnswerForFormField({
        sampling: ctx.sampling,
        formField,
        jobContext,
        applicantContext
      });
      if (answer && answer.length > 0) {
        fields.push({ kind: 'text_by_name', name: formField.name, value: answer });
        sampledFields.push(formField.name);
      }
    }
  }
```
